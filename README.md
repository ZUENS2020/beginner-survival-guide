## 核心原则

1. **显式优于隐式** — 永远显式指定，不要依赖默认值
2. **先搞清楚"不能做什么"** — 限制比能力更重要
3. **中间件是 bug 高发区** — 翻译必须完整
4. **精准比全面重要** — 少而准 > 多而乱
5. **自愈设计** — 假设进程会意外终止
6. **不要想当然** — 5 分钟读文档省 5 小时调试
